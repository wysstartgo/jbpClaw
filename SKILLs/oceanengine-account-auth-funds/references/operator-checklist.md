# 账户、鉴权与资金执行检查清单

## 执行前

- 明确用户目标是查询、分析、还是执行写操作。
- 确认当前请求是否完全落在本域；若跨域，先拆分再处理。
- 确认目标账户、对象、时间范围或资产范围。
- 先读取 `read-tools.json` 建立上下文，不直接进入写操作。

## 执行中

- 仅在 `tool-range.json` 白名单内选工具。
- 如需变更，优先从最小对象、最小批量开始。
- 命中 `risky-tools.json` 时，先向用户确认影响范围。
- 若实际结果与预期不一致，先停在当前域内做状态复核。

## 执行后

- 输出已调用工具、核心结果和异常点。
- 若需要后续域继续处理，按 `handoffs.md` 明确交接。
- 对写操作补充回滚难度、持续观察点和后续建议。

## 本域特别提醒

- 默认先查账户、预算、余额、资质状态，再决定是否执行更新类操作。
- 涉及预算更新、钱包预算提交、安全积分或资质提交时，先明确影响账户和变更范围。
- OAuth / Access-Token 相关问题优先判断权限前置是否满足，避免盲目调用下游业务接口。

## 推荐先走的工作流

- `账户体检`：快速了解一个广告主账户的基础状态和潜在风险。
- `鉴权前置检查`：判断当前 token 与账户关系是否足以支撑后续业务调用。
